
# Tailor Storyboard Data Model & Structure

This document outlines the data structure and Firestore collection schema for the **Tailor Storyboard** feature. This can be used to implement frontend displays and manage data consistency.

## 1. Firestore Collection

**Collection Name:** `tailor_storyboards`

- All storyboard documents are stored in this root-level collection.
- Documents are identified by an auto-generated Firestore ID.

## 2. Data Model (`TailorStoryboard`)

The core data type is `TailorStoryboard`.

| Field | Type | Description |
| :--- | :--- | :--- |
| `id` | `string` | The unique document ID (auto-generated by Firestore). |
| `title` | `string` | The display title of the promotion/storyboard (e.g., "Summer Collection"). |
| `tailorId` | `string` | The ID of the tailor/vendor this storyboard belongs to. Links to `tailors` collection. |
| `tailorName` | `string` | Cached display name of the tailor (Brand Name or Full Name). |
| `tailorDescription` | `string` | Curated bio/story about the tailor. This is editable marketing copy. |
| `tailorLogo` | `string` (optional) | URL to the tailor's logo image. |
| `productIds` | `string[]` | Array of product IDs that belong to this storyboard. Links to `tailor_works` collection. |
| `active` | `boolean` | Status flag. `true` if the storyboard is live/visible. |
| `createdBy` | `string` | ID of the marketing user who created this storyboard. |
| `createdAt` | `Timestamp` | Firestore Timestamp of creation. |
| `updatedAt` | `Timestamp` | Firestore Timestamp of last update. |
| `productsCount` | `number` (optional) | Cached count of products in the storyboard. |
| `thumbnailUrl` | `string` (optional) | URL of the first product image, used as a preview thumbnail. |
| `previewImage` | `string` (optional) | **New:** Main front-facing image URL for list views/cards. |
| `bannerImage` | `string` (optional) | **New:** Secondary banner image URL for detailed views. |

## 3. Relationships & Frontend Implementation Guide

### A. Fetching Products
The `productIds` array contains ID references to the products. To display the full product grid for a storyboard:
1.  **Retrieve Storyboard**: Get the `TailorStoryboard` document.
2.  **Fetch Products**: Query the `tailor_works` (products) collection using the `tailorId` from the storyboard.
    *   *Note: Since the feature now enforces showing ALL products for a tailor, it is more efficient to query products where `tailor_id == storyboard.tailorId` rather than fetching by individual IDs in the `productIds` array.*

### B. Fetching Tailor Details
While `tailorName` and `tailorLogo` are cached in the storyboard for quick display, you may want to fetch the latest full tailor profile:
1.  Use `storyboard.tailorId` to fetch the document from the `tailors` collection.

## 4. TypeScript Interface

Use this interface in your frontend code for type safety:

```typescript
import { Timestamp } from 'firebase/firestore';

export interface TailorStoryboard {
  id: string;
  title: string;
  tailorId: string;
  tailorName: string;
  tailorDescription: string;
  tailorLogo?: string;
  productIds: string[];
  active: boolean;
  createdBy: string;
  createdAt: Timestamp;
  updatedAt: Timestamp;
  productsCount?: number;
  thumbnailUrl?: string;
  previewImage?: string;
  bannerImage?: string;
}
```
